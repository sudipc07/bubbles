## 1. Product Brief

**What it is.** A web-based platform that autonomously generates LinkedIn and Instagram content for multiple products using a multi-agent AI pipeline. The system writes posts, designs carousels, packages assets per channel, and queues finished work for a human to publish manually.

**Why it exists.** Running a product brand's social presence well is expensive and slow. Off-the-shelf AI writers produce generic copy that ignores audience, brand, and platform context. This platform produces on-brand, audience-aware content at scale, with humans only in the loop for review and the actual posting click.

**Who uses it.** Product marketers, founders, and operators running one or more brand social accounts who want consistent, high-quality content without writing it themselves.

**Why it works.** Two agent layers — a setup-time wizard that captures product, audience, voice, and design once; and a runtime pipeline of specialist agents that produces posts on schedule. Each post is generated by a small team of agents (planner, writer, critic, designer, packager) rather than a single generalist.

## 2. Platform Users (Roles)

| Role | Capabilities |
|------|--------------|
| **Owner** | Create project, configure, invite/remove members, delete project, change role, billing (v2) |
| **Member** | Run wizard, edit personas/themes/brand kit, review drafts, approve/edit/reject, mark-as-posted |

Two roles only for MVP. Viewer role deferred to v2.

A user can be Owner of some projects and Member of others. A project must have at least one Owner.

---

## 3. Success Metrics (v1)

- **Time saved per post.** Human time per published post < 2 minutes (review + manual publish).
- **First-pass approval rate.** ≥ 80% of generated drafts approved without major edits.
- **Throughput.** A project ships ≥ 5 posts per week with zero human writing.
- **Multi-project clarity.** Platform supports ≥ 3 concurrent projects per user without confusion (measured by user testing).
- **Quality gate hit rate.** Empathy and brand-safety gates catch ≥ 95% of problem drafts before queue.

## 4. Functional Spec

### 4.1 Top-level navigation

- **Projects list** — landing page after login
- **Per-project sidebar** — Dashboard, Drafts queue, History, Personas, Themes, Brand kit, Members, Settings
- **Global** — Profile (notification preferences), Sign out

### 4.2 Key screens

**Projects list.** Cards per project showing name, enabled channels, last post date, count of drafts pending review. Primary action: "New Project" (kicks off wizard).

**Dashboard.** Per project. Shows: what's ready to publish, what's queued for generation, recent engagement summary (last 30 days), top-performing persona, helpfulness ratio gauge.

**Drafts queue.** List of pending drafts. Each row: persona, format, theme, generated-at, preview thumbnail, status. Per-row actions: Approve, Edit, Regenerate, Reject.

**Post detail / preview.** Opens from a queue row. Shows the full draft: post copy, carousel slides rendered, channel-specific bundles (LinkedIn PDF, Instagram PNG set). Actions: Copy caption, Download PDF, Download ZIP of PNGs, Mark as posted (with optional URL paste-back), Regenerate, Reject.

**Personas tab.** 5 cards per project showing name, voice spec, content focus, format mix (carousel/single image %), status (active/paused), 30-day performance. Per-card actions: Edit, Pause, Regenerate, Delete.

**Themes tab.** List of themes, frequency weights, enabled toggle. Add custom theme. Edit existing.

**Brand kit tab.** Shows logo, extracted palette (primary/secondary/accent), chosen fonts, sample slide template preview. Allow re-upload of logo or manual override of palette/fonts (advanced).

**Members tab.** List of project members with role. Owner can invite by email, change roles, remove.

**Settings tab.** Per project: channels enabled (LinkedIn, Instagram), posting schedule (auto / fixed slots), approval gate strictness (all drafts / only flagged), helpfulness ratio target (default 9:1), notification mode override.

**Profile.** Per user: name, email, notification mode (push / digest-daily / digest-weekly / none), default landing project.

### 4.3 Notification modes

- **Push.** Each draft generation triggers in-app notification (and email if enabled). Default for users who want it surfaced.
- **Digest.** Daily or weekly summary email with all pending drafts.
- **None.** User checks dashboard on their own schedule.

Default: Push. Configurable per user in Profile. Project Owners can recommend a project-wide default for invitees.

## 5. Setup Wizard Flow

Choice-driven. No free chat. Each step renders AI-generated options as cards; user picks. Free text only for corrections.

Steps:

1. **Project basics.** Project name (free text). Channels to enable (toggle: LinkedIn, Instagram, both). Channel handles/URLs.
2. **Product input.** Paste PRD or product description (free text or file upload). Upload logo (PNG/SVG/JPG).
3. **Confirm understanding.** AI's parsed product summary as a card. Actions: "Looks right" or "Edit" (free text correction).
4. **Audience selection.** 4 audience profile cards generated. Pick primary, optional secondary. "Regenerate options" if none fit.
5. **Voice direction.** 4 voice direction cards (e.g., "Warm and direct", "Authoritative and data-led"). Single pick.
6. **CTA style.** 4 CTA strategy cards (e.g., "Soft signup hook", "Free resource lead", "Direct sell"). Single pick.
7. **Posting cadence.** 3 cards (daily / MWF / Tue-Thu). Single pick.
8. **Brand kit preview.** Auto-generated from logo: palette + fonts + sample slide template. "Looks good" or "Try again".
9. **Personas review.** 5 personas generated, each card shows name, voice spec, content focus, format mix. Per card: Use, Swap, Regenerate.
10. **Themes review.** 8-10 themes generated as toggle cards. Toggle on/off, add custom theme.
11. **Sample posts review.** 2 sample posts per persona (10 total) generated. Thumbs up/down per post — becomes voice training data for runtime.
12. **Go live.** Summary screen. Confirm activation. First scheduled draft appears in queue per cadence.

Wizard is **re-runnable per project**. Owner can re-run any step (e.g., regenerate personas after 3 months) from the relevant tab.

## 6. Multi-Agent Workflow

### 6.1 Setup-time agents (run during wizard)

| Agent | Job |
|-------|-----|
| **Parser** | Reads PRD; extracts product features, audience hints, value props, brand cues |
| **Audience Generator** | Proposes 4 audience profile cards |
| **Voice Generator** | Proposes 4 voice direction cards |
| **Persona Generator** | Proposes 5 personas with voice spec, content focus, format mix |
| **Theme Generator** | Proposes 8-10 content themes |
| **Brand Kit Generator** | Extracts palette from logo, picks complementary fonts, renders template preview |
| **Sample Generator** | Drafts 2 sample posts per persona; thumbs-up posts become voice training set |

### 6.2 Runtime agents (run on schedule)

| Agent | Job |
|-------|-----|
| **Planner** | Picks persona + theme + format for next post, given recent activity and helpfulness ratio |
| **Researcher** | Pulls relevant external context (news, trends, examples of successful posts in the niche) |
| **Writer** | Drafts post content in chosen persona's voice. Single block for text post; structured slide spec for carousel |
| **Empathy Critic** | Reads as a member of the target audience. Scores helpful / performative / tone-deaf. Returns notes if fail |
| **Brand Safety** | Checks helpfulness ratio over rolling 30 days, uniqueness vs last N posts, banned patterns, hallucinated stats |
| **Editor** | Incorporates critic notes, polishes for format |
| **Designer** | Renders carousel slides as branded PNGs using brand kit; renders single image; skips for text-only |
| **LinkedInPackager** | Produces LinkedIn bundle: PDF (with document title) + caption (≤3000 chars, 3-5 hashtags) + suggested publish time |
| **InstagramPackager** | Produces Instagram bundle: PNG sequence + caption (≤2200 chars, 15-30 hashtags) + suggested publish time |
| **Analyst** | Runs 48h after marked-as-posted. Pulls engagement (manual entry in MVP). Tags learnings into knowledge base |

**Pipeline order:**

Planner → Researcher → Writer → Empathy Critic → Brand Safety → Editor → Designer → [LinkedInPackager ‖ InstagramPackager run in parallel] → drafts queue → human → Analyst (48h later)

### 6.3 Quality gates

- **Empathy gate.** Pass: helpful. Fail-soft: performative — regenerate once with critic notes. Fail-hard: tone-deaf — regenerate from Writer with full reset.
- **Brand safety gate.** Hard fail on banned patterns or hallucinated stats; draft is killed, not queued.
- **Uniqueness gate.** Fail if similarity to any of last 10 posts above threshold; regenerate with diversity hint.
- **Helpfulness ratio gate.** Over rolling 30 days, max 1 promo-style post per 9 utility posts. Brand Safety blocks the 10th promo.

## 7. Data Model (logical)

Entities and key fields. Implementation choices (DB engine, ORM, etc.) left to the build.

- **User** — id, name, email, notification_mode, default_project_id
- **Project** — id, name, description, parsed_product_summary, logo_asset_id, palette, fonts, status (draft/active/paused), helpfulness_ratio_target, posting_schedule
- **ProjectMember** — user_id, project_id, role (Owner / Member)
- **Channel** — id, project_id, type (LinkedIn / Instagram), handle_or_url, enabled
- **Persona** — id, project_id, name, voice_spec, content_focus, format_mix_carousel_pct, format_mix_single_pct, status (active/paused), training_samples_ids
- **Theme** — id, project_id, name, description, frequency_weight, enabled
- **Draft** — id, project_id, persona_id, theme_id, format (carousel / single_image), content_text, slide_structure (JSON if carousel), critique_notes, status (pending / approved / rejected / posted), created_at
- **PostBundle** — id, draft_id, channel_id, caption, asset_urls (array), title (for LinkedIn PDF), status (ready / posted), posted_at, public_url
- **EngagementMetric** — id, post_bundle_id, impressions, likes, comments, shares, pulled_at, source (manual / auto)
- **Asset** — id, project_id, type (logo / slide / pdf / bundle_zip), url, metadata

## 8. MVP Scope

**In scope:**

- Multi-project, multi-user with Owner/Member roles
- 12-step setup wizard
- 9-agent runtime pipeline + 7 setup-time agents
- LinkedIn + Instagram as channels
- Carousel + single image formats
- Drafts queue with approve / edit / regenerate / reject
- Mark-as-posted with optional URL paste-back
- Manual engagement entry
- Brand kit auto-generated from logo
- Notification modes (push / digest / none)
- Per-project dashboard with key metrics

**Out of scope (deferred to v2+):**

- Video format (Reels, LinkedIn video)
- Automatic posting via channel APIs (no LinkedIn / Instagram API integration in MVP)
- Auto-pull engagement metrics
- Custom personas defined from scratch (only generator-then-edit in MVP)
- Channels beyond LI + Insta (X, Threads, Bluesky, etc.)
- Billing / subscription / team management features
- Rich analytics dashboard with charts
- Slack / Teams notifications
- A/B testing variants per post
- Custom voice training via uploaded exemplar posts
- Viewer role

## 10. Acceptance Criteria (v1 done)

A v1 build is complete when:

1. A new user can sign up and complete the setup wizard end-to-end for a project in under 15 minutes.
2. The runtime pipeline produces its first draft within 5 minutes of project activation.
3. Generated drafts include all of: post copy; carousel PNGs if carousel format; LinkedIn PDF if LinkedIn enabled; Instagram PNG set if Instagram enabled; suggested publish time.
4. A user can approve, edit, regenerate, or reject any draft; rejected drafts trigger regeneration.
5. Marking-as-posted records timestamp; user can optionally paste the public post URL.
6. Manual engagement entry updates the post's metrics and feeds the Analyst.
7. Two users with different roles can collaborate on the same project without permission errors.
8. Notifications fire per each user's preference (push / digest / none).
9. Brand Safety blocks a 10th promo-style post in a 30-day window when the helpfulness ratio is breached.
10. Empathy gate blocks ≥ 95% of tone-deaf drafts before they reach the queue (measured against a curated test set).
11. The platform runs at least 3 concurrent projects with distinct brand kits, personas, and themes without bleed across projects.
